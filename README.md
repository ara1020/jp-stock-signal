# 日本株シグナルウォッチャー

日本株のウォッチリストを登録し、テクニカル指標に基づく売買タイミングの目安を表示する Web アプリです。
データは Yahoo Finance から取得します。銘柄コードは英字を含む新形式(例: 285A)にも対応しています。

主な機能:

- **ウォッチリスト評価**: 登録した銘柄ごとに、下記のルールベース採点でスコアと判定
  (強い買い候補〜強い売り候補)を根拠付きで表示。2024年1月以降の新規上場銘柄
  (英字入りコード: 285A キオクシア、215A タイミー等)にも対応。
- **買い候補スクリーニング**: 東証の主要約60銘柄(2024年以降のIPO銘柄を含む)を採点し、
  上位8銘柄を根拠付きで提示。株価上限(円)を指定して「XXXX円以下の銘柄だけ」に絞り込むことも可能。

採点に使うルール(すべて画面上に根拠として表示されます):

| ルール | 出典となる考え方 |
|---|---|
| 25日線の位置と傾きによるトレンド判定 | ダウ理論 / トレンドフォロー |
| SMA5/25 のゴールデンクロス・デッドクロス | 移動平均分析 |
| RSI(14) の過熱・売られすぎ判定 | ワイルダーの RSI |
| 直近25営業日リターン | モメンタム効果 |
| 出来高の増加 | 出来高分析 |
| 6ヶ月高値圏への接近 | オニールの新高値ブレイクアウト |
| 25日線からの上方乖離 | グランビルの法則 |

> **注意**: 採点は公開されたテクニカル指標による機械的なルールであり、投資助言ではありません。
> 業績・割安度などのファンダメンタルズやニュースは考慮していません。将来の値動きを保証するものでも
> ありません。投資判断は自己責任でお願いします。

## ローカルで動かす

Python 3.9 以降が必要です(追加ライブラリのインストールは不要)。

```bash
python server.py
```

`http://127.0.0.1:8787` をブラウザで開いてください。

## 仕組み

- `server.py`: 静的ファイル配信 + Yahoo Finance API へのプロキシ(ブラウザから直接叩くと CORS で
  弾かれるため、同一オリジンのサーバーを経由させている)。
- `public/`: フロントエンド一式(HTML/CSS/JS)。SMA・RSI の計算とシグナル判定、チャート描画はすべて
  `public/app.js` 内の素の Canvas API で行っている。

## デプロイ(Render.com)

このリポジトリには `render.yaml` を同梱しているので、Render の Blueprint 機能を使えばそのままデプロイできます。

1. [Render](https://render.com) にサインアップし、GitHub アカウントと連携する。
2. 「New +」→「Blueprint」からこのリポジトリを選択する。
3. デプロイが完了すると `https://<サービス名>.onrender.com` の URL が発行される。

無料プランは一定時間アクセスがないとスリープし、次回アクセス時に数十秒の起動遅延が発生します。
